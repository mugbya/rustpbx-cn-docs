# RustPBX 是什么

RustPBX 是一个用 Rust 语言编写的高性能、软件定义的 PBX（Private Branch Exchange，专用分组交换机）系统。你可以把它理解为一个“AI 原生的现代电话交换系统”，旨在替代传统的 PBX 系统（如 FreeSWITCH）。

它的核心设计目标是通过开放的标准协议（HTTP/WebSocket），让开发者能像编写普通 Web 应用一样，用任何编程语言来控制电话呼叫、媒体流和事件

### 项目地址

[rustpbx](https://github.com/restsend/rustpbx) 欢迎一键三连，支持 Star、Watch、Fork。


## 核心特性

RustPBX 的主要特性可以分为以下几个层面：

| 特性模块           | 具体功能                                                     |
| :----------------- | :----------------------------------------------------------- |
| **SIP & 媒体核心** | 提供完整的 SIP 协议栈（支持 UDP/TCP/WS/TLS/WebRTC），内置 RTP 媒体代理、NAT 穿透，并支持通话录音、TLS/SRTP 加密及自动证书管理。 |
| **AI 与语音能力**  | 集成语音活动检测（VAD）、语音转文字（ASR）和文字转语音（TTS），支持对接 OpenAI 等大语言模型（LLM），可快速构建 AI 语音助手。 |
| **可编程接口**     | 所有呼叫控制通过 **HTTP/WebSocket/Webhook** 对外开放。例如，每次呼叫都可触发你的 HTTP 服务，由你返回 JSON 格式的路由决策。 |
| **管理与运维**     | 提供内置的 Web 控制台和 WebRTC 电话，支持 MySQL、PostgreSQL、SQLite 多种数据库，并提供 Prometheus 指标和 OpenTelemetry 用于监控。 |


## 技术架构与设计理念

RustPBX 的架构设计有几点特别之处：

高性能：得益于 Rust 语言的高效特性，它性能出色。官方数据显示，在处理 5000 路并发通话时，仅需约 3.8 个 CPU 核心，且无数据包丢失。

软件定义：它的核心设计是“软件定义”。所有的呼叫逻辑都通过标准 HTTP API 暴露，这意味着你无需修改 RustPBX 本身的代码或重启服务，就可以通过更新自己的 Web 服务来改变呼叫流程。

模块化：主要功能分为 SIP 代理和 PBX 核心，并且将“语音 Agent”等高级功能分离为独立项目，保持核心的精简和专注。


## 适用场景

基于以上特性，RustPBX 特别适合以下场景：

下一代呼叫中心：其 AI 原生架构非常适合构建智能客服、语音机器人等应用。

AI 语音应用开发：它为开发者提供了一个强大的平台，可以快速将 ASR、TTS、LLM 等能力集成到电话系统中。

需要高并发、高可控性的 VoIP 系统：对于性能和可编程性有较高要求的企业通信系统，RustPBX 是一个现代的、强有力的选项。
